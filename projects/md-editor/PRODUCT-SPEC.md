# MyMD — 跨平台 Markdown 编辑器

> 此文档是 AI 执行基准，不是给人阅读的文档。

## 1. 产品概述

**产品定位**: 面向产品经理和专业写作者的专业 Markdown 编辑器，强调"即时渲染"体验（无需切换视图，编辑即呈现），支持代码高亮、Mermaid 图表、多格式导出。

**核心差异化**: 极简沉浸式编辑 + 实时渲染 + 本地文件夹管理 + 无需云端账户

---

## 2. 目标用户

| 维度 | 描述 |
|------|------|
| 角色 | 产品经理、技术文档写作者 |
| 场景 | 日常笔记、会议记录、产品文档、技术博客 |
| 痛点 | 现有工具要么太简单（无代码/图表），要么太重（需云端/账户），要么预览割裂（编辑/预览分离） |
| 设备 | Windows/macOS/Linux 桌面端 |

---

## 3. 功能规格

### 3.1 核心功能

#### Feature-001: 即时渲染编辑器
- **描述**: Typora 风格的即时渲染编辑体验，Markdown 语法在输入时即时转换为富文本显示
- **输入**: 用户键盘输入 Markdown 文本
- **输出**: 实时渲染的富文本视图，光标处可继续编辑
- **用户故事**: "我希望在写作时能看到最终的排版效果，而不是在编辑和预览之间切换"
- **验收标准**:
  - [ ] 输入 `# 标题` 后立即显示大字标题样式（非源码态）
  - [ ] 输入 `**粗体**` 后立即显示粗体效果
  - [ ] 光标可在渲染后的内容中移动并继续编辑
  - [ ] 支持源码模式切换（快捷键 Ctrl+/）
- **优先级**: P0
- **技术说明**: 基于 Tauri + WebView，使用 `markdown-it` 或 `marked` 解析，配合 contenteditable 或自定义渲染层实现即时渲染

#### Feature-002: 代码高亮
- **描述**: 编辑器和渲染视图中对代码块进行语法高亮
- **输入**: 带有语言标识的代码块（如 ```javascript）
- **输出**: 带语法高亮的代码显示
- **用户故事**: "我需要写技术文档，代码块必须有高亮"
- **验收标准**:
  - [ ] 支持 20+ 常用语言（JS/TS/Python/Go/Rust/Java/C++/SQL/Bash 等）
  - [ ] 代码块有复制按钮
  - [ ] 行号显示
- **优先级**: P0
- **技术说明**: 使用 `highlight.js` 或 `Prism.js`

#### Feature-003: Mermaid 图表
- **描述**: 在 Markdown 中嵌入 Mermaid 语法，实时渲染为图表
- **输入**: Mermaid 语法文本（flowchart、sequence、gantt 等）
- **输出**: 渲染后的 SVG/Canvas 图表
- **用户故事**: "我需要在文档中画流程图，直接用代码不用截图"
- **验收标准**:
  - [ ] 支持 flowchart、sequence diagram、gantt chart
  - [ ] 图表在渲染视图实时更新
  - [ ] 支持代码块内联渲染（非外置图片）
- **优先级**: P0
- **技术说明**: 使用 `mermaid.js` 库

#### Feature-004: 本地文件夹管理
- **描述**: 直接打开本地文件夹作为工作区，在编辑器中管理文件树
- **输入**: 用户选择本地文件夹路径
- **输出**: 侧边栏显示文件夹树结构，支持新建/删除/重命名文件
- **用户故事**: "我的文档都存在本地目录里，不需要云端同步"
- **验收标准**:
  - [ ] 文件树展示，支持展开/折叠
  - [ ] 新建文件（.md）
  - [ ] 删除文件（确认提示）
  - [ ] 重命名文件
  - [ ] 支持拖拽移动文件
- **优先级**: P0
- **技术说明**: Tauri 的 fs 系统调用，本地文件读写

#### Feature-005: 多格式导出
- **描述**: 将 Markdown 文档导出为多种格式
- **输入**: 当前打开的 .md 文件
- **输出**: PDF/HTML/Word(EPUB) 等格式的文件
- **用户故事**: "我写的文档需要分享给其他人，可以导出不同格式"
- **验收标准**:
  - [ ] 导出为 PDF（保留样式）
  - [ ] 导出为 HTML（自包含单文件）
  - [ ] 导出为 Word（.docx）
  - [ ] 导出为 EPUB（电子书格式）
  - [ ] 批量导出（当前文件夹所有 .md）
- **优先级**: P1
- **技术说明**: PDF 使用 `wkhtmltopdf` 或 Tauri PDF 插件；HTML 使用 markdown-it 输出；Word 使用 `docx` 库；EPUB 使用 `epub-gen`

#### Feature-006: 外观定制
- **描述**: 编辑器外观（主题、字体、字号）可自定义
- **输入**: 用户设置偏好
- **输出**: 应用新的视觉样式
- **用户故事**: "我希望编辑器和预览的字体大小适合我的视力"
- **验收标准**:
  - [ ] 深色/浅色主题切换
  - [ ] 编辑器字体选择（等宽字体列表）
  - [ ] 字号大小调节
  - [ ] 主题色自定义
- **优先级**: P1
- **技术说明**: CSS 变量 + localStorage 持久化

#### Feature-007: 搜索与替换
- **描述**: 在当前文档或所有打开的文件中搜索文本
- **输入**: 搜索关键词
- **输出**: 搜索结果列表，支持跳转
- **用户故事**: "我需要在长文档中快速找到某个关键词"
- **验收标准**:
  - [ ] Ctrl+F 当前文档搜索
  - [ ] 搜索结果高亮
  - [ ] 支持正则表达式
  - [ ] 支持替换（单处/全部）
- **优先级**: P1

#### Feature-008: 自动保存
- **描述**: 编辑时自动保存，防止数据丢失
- **输入**: 用户输入
- **输出**: 定时自动保存到磁盘
- **用户故事**: "我不想因为突然崩溃丢失写作内容"
- **验收标准**:
  - [ ] 每 30 秒自动保存（可配置）
  - [ ] 崩溃恢复（重新打开时恢复未保存内容）
  - [ ] 保存状态指示（已保存/保存中）
- **优先级**: P2

#### Feature-009: 标签与元数据
- **描述**: 为文档添加标签，便于分类管理
- **输入**: 用户为文档添加标签
- **输出**: 按标签筛选文档
- **用户故事**: "我的笔记需要打标签方便回顾"
- **验收标准**:
  - [ ] 文档头部 YAML 元数据（tags）
  - [ ] 侧边栏标签筛选
  - [ ] 标签自动补全
- **优先级**: P2

#### Feature-010: 大纲视图
- **描述**: 文档标题层级结构概览
- **输入**: 当前文档的 Markdown 内容
- **输出**: 可点击的大纲树形结构
- **用户故事**: "我需要快速导航到文档的某个章节"
- **验收标准**:
  - [ ] 从 H1-H6 提取标题生成大纲
  - [ ] 点击跳转对应位置
  - [ ] 大纲随内容滚动同步高亮
- **优先级**: P1

## 4. 技术架构

### 4.1 技术选型

| 组件 | 技术 | 说明 |
|------|------|------|
| 框架 | **Tauri 2.x** | 跨平台（Win/Mac/Linux），体积小（<10MB），原生性能 |
| 前端 | **React 18** 或 **Solid.js** | 响应式 UI |
| 渲染引擎 | markdown-it | Markdown 解析，支持插件扩展 |
| 代码高亮 | highlight.js | 成熟，支持 190+ 语言 |
| 图表 | mermaid.js | 支持多种图表类型 |
| 状态管理 | Zustand 或 Jotai | 轻量 |
| 样式 | Tailwind CSS | 快速开发 |

### 4.2 项目结构（提议）

```
src/
├── main.tsx              # Tauri 入口
├── App.tsx               # 主应用
├── components/
│   ├── Editor/           # 即时渲染编辑器
│   ├── Sidebar/         # 文件树 + 标签
│   ├── Toolbar/         # 工具栏
│   └── Export/          # 导出面板
├── lib/
│   ├── markdown/         # markdown-it 配置，插件
│   ├── mermaid/          # mermaid 初始化
│   └── export/           # 导出功能
├── stores/               # Zustand stores
└── styles/               # 全局样式
```

### 4.3 平台支持优先级

1. **Windows** (Primary) — 主要开发环境
2. **macOS** (Secondary) — Electron 模式打包
3. **Linux** (Tertiary) — AppImage/DEB

## 5. 非功能需求

| 需求 | 指标 |
|------|------|

## 6. 边界定义

### 明确不支持

- ❌ 云端同步/账户系统（纯本地）
- ❌ 协作文档（单用户）
- ❌ 插件生态（v1.0）
- ❌ 移动端（v1.0）

### MVP 范围（v1.0）

1. 即时渲染编辑器（核心）
2. 代码高亮
3. Mermaid 图表
4. 本地文件夹管理
5. 深色/浅色主题
6. PDF/HTML/Word 导出
